# Episteme 用户功能 PRD

## 1. 产品定位

Episteme 是面向高频论文阅读者的本地优先研究工作台。用户可以在同一个桌面应用中完成论文发现、导入、整理、阅读、选区追问、带引用的 Agent 对话、研究笔记、生成图像预览和会话恢复。

本 PRD 只描述用户功能、交互效果和验收标准，不规定具体技术实现。后续使用任何新框架重构时，应保持这些用户可感知能力和工作流连续性。

## 2. 目标用户

- 研究者、工程师、学生、审稿人：需要持续阅读和管理大量论文。
- 论文追踪者：需要从 arXiv 发现新论文，并快速判断是否值得阅读。
- AI 辅助阅读用户：希望围绕真实 PDF、选区、引用和本地会话与 Agent 协作。
- 本地工作流用户：希望论文、缓存、笔记、会话和生成物可在本机长期保留，并可被外部 Agent 读取。

## 3. 产品目标

- 让用户把论文发现、文库整理、PDF 阅读和 Agent 对话放在一个连续工作流中完成。
- 让用户始终知道当前对话、笔记和生成物对应哪篇论文、哪组论文、哪段原文。
- 让 arXiv 发现结果从“论文列表”变成“可筛选、可增强、可保存、可打开阅读”的研究入口。
- 让 Agent 运行结果可检查、可恢复、可回跳到原文，不把回答变成脱离来源的聊天记录。
- 让用户能根据自己的阅读习惯配置语言、排序、相似度、Agent runtime、Prompt、字体和缓存。

## 4. 范围

### 4.1 本 PRD 覆盖

- 主导航与信息架构。
- 本地论文文库。
- arXiv Discover 与 arXiv 搜索。
- PDF 阅读器。
- 阅读会话、聊天、Agent 运行、终端与笔记。
- 生成图像展示与预览。
- 最近会话。
- 设置与偏好。
- 外部 Agent / MCP 可感知能力。
- 用户可见状态、错误处理和验收标准。

### 4.2 本 PRD 不覆盖

- 具体框架、语言、数据库、API、线程模型、缓存格式、打包方式。
- 内部模块划分。
- 后端服务设计。
- 视觉设计规范的像素级细节。

## 5. 信息架构

应用应提供以下一级入口：

- Library：管理本地论文、文件夹、标签、笔记和批量操作。
- 探索：按日期、关键词、类别和相似度发现近期 arXiv 论文。
- 搜索：按 arXiv 查询语法、年份、类别和排序搜索更大范围的论文。
- Reader：阅读当前论文或多论文会话，并与 Agent 协作。
- Recent Conversations：恢复历史研究会话。
- Settings：配置语言、阅读体验、arXiv、增强、Agent、Prompt、插件和缓存。

导航应允许用户在整理、发现、阅读和设置之间快速切换，并保持当前上下文尽量不丢失。

## 6. 核心用户流程

### 6.1 从本地 PDF 开始阅读

1. 用户在 Library 导入一个或多个 PDF，或把 PDF 拖入文库。
2. 应用解析论文标题、作者、年份、来源等可展示元数据。
3. 用户可把论文放入文件夹、添加标签、加星、写笔记。
4. 用户点击 Read 进入阅读器。
5. 用户在 PDF 中选中文本，右侧对话区出现“正在回复原文”的提示。
6. 用户提问后，Agent 基于当前论文上下文回答。
7. 回答中的引用可让用户跳回原 PDF 位置。

成功效果：用户可以从一个 PDF 进入可追问、可引用、可记录的研究会话。

### 6.2 从 arXiv 发现论文并保存

1. 用户进入探索页，设置日期范围、类别、关键词和相似度来源。
2. 应用显示论文卡片网格，展示标题、作者、类别、摘要/增强信息、资源链接、相似度和保存状态。
3. 用户可筛选已处理/未处理/失败、新论文/已在文库、有代码或项目链接、相似度等级、标签。
4. 用户可对结果执行处理动作：翻译标题、生成中文总结/贡献/标签/链接、下载 PDF 并生成缩略图。
5. 用户可预览论文图片或 PDF 缩略图，打开论文阅读，或保存到指定文件夹。

成功效果：用户能把 arXiv 结果变成可判断、可阅读、可归档的本地论文。

### 6.3 精确 arXiv 搜索

1. 用户进入搜索页。
2. 用户输入 arXiv 查询，选择排序方式和升降序。
3. 用户可设置起止年份，以及“至少包含”的 arXiv 类别。
4. 用户可在搜索结果中按结果类别和文库状态继续筛选。
5. 用户可对搜索结果执行与 Discover 一致的处理、保存和打开动作。

成功效果：用户能用结构化检索找到目标论文，并接入同一套保存与阅读流程。

### 6.4 多论文阅读和对话

1. 用户在文库中选择多篇论文，点击 Read 或 Chat。
2. 阅读器顶部展示当前会话包含的论文标签。
3. 用户可在同一会话中添加或移除论文。
4. 用户可切换当前阅读论文，但会话仍保留同组论文上下文。
5. Agent 回答应能区分论文来源，并支持引用回跳。

成功效果：用户可以围绕一组论文进行对比、综述或批判性阅读，而不是被迫逐篇孤立对话。

### 6.5 恢复历史研究

1. 用户进入 Recent Conversations。
2. 列表展示历史会话标题、更新时间、包含论文数量和主要论文信息。
3. 用户选中会话后，详情面板展示会话包含的论文。
4. 用户点击 Open Session 回到阅读器和会话上下文。

成功效果：用户可以继续过去的研究现场，避免重新找论文、重建上下文和重复提问。

## 7. 功能需求

### 7.1 主导航

需求：

- 用户应能从侧边栏进入 Library、探索、搜索、Settings 和 Recent Conversations。
- 当前入口应有明确选中态。
- 从文库或最近会话打开论文后，应进入 Reader。
- 在 Reader 中返回 Library 时，应尽量保留用户之前的文库筛选和选择上下文。

验收标准：

- 用户能在所有一级入口之间切换。
- 入口切换不应清空用户已设置的搜索词、筛选条件、阅读会话或正在编辑的草稿，除非该动作本身明确要求重置。

### 7.2 本地论文文库

需求：

- 用户可导入单个或多个 PDF。
- 用户可拖拽 PDF 到文库区域完成导入。
- 用户可配置监听文件夹，并手动扫描监听文件夹。
- 用户可移除监听文件夹；已导入论文不应因此消失。
- 用户可通过粘贴 arXiv ID、链接、PDF 链接或包含 ID 的任意文本，批量加入 arXiv 论文。
- 文库列表应展示论文标题、作者、年份、来源、加星状态、缩略图、文件夹和标签摘要。
- 用户可按标题、作者、标签、类别、年份或来源搜索论文。
- 用户可按文件夹筛选，也可选择是否包含子文件夹。
- 用户可按标签筛选。
- 用户可按加入时间、标题、arXiv ID 排序，并切换升降序。
- 加星论文应在排序中更容易被看到。
- 用户可选中单篇论文查看详情。
- 用户可多选论文并执行批量 Read、Chat、Copy、Tag、Delete、Clear。
- 删除论文前必须让用户确认，并清楚说明影响。

效果：

- 文库应保持适合扫描的密度，方便用户在大量论文中快速定位目标。
- 用户能把文库当成长期研究收藏，而不是一次性文件列表。

验收标准：

- 导入后的论文可被搜索、筛选、阅读和加入会话。
- 多选批量操作只影响用户选中的论文。
- 清空筛选后，用户能回到完整论文列表。

### 7.3 文件夹管理

需求：

- 用户可创建顶层文件夹和子文件夹。
- 用户可重命名文件夹。
- 用户可删除文件夹；删除文件夹应移除文件夹和其子文件夹及归属关系，但不删除论文。
- 用户可折叠/展开文件夹树。
- 用户可 pin / unpin 文件夹，使重要文件夹更容易访问。
- 用户可通过拖拽调整文件夹父子关系和顺序。
- 用户可把论文拖入文件夹。
- 用户可把文件夹拖到顶层。

效果：

- 文件夹树应表现为研究主题结构，支持长期整理和快速收纳。

验收标准：

- 文件夹层级变化后，文库筛选结果立即反映新的归属关系。
- 删除文件夹不会误删论文。
- 拖拽过程中应让用户看清落点和结果。

### 7.4 标签管理

需求：

- 用户可创建、重命名、删除标签。
- 用户可在论文详情中给单篇论文添加或移除标签。
- 用户可批量给多篇论文添加标签。
- 删除标签应从所有论文移除该标签，但不删除论文。
- 标签筛选应显示与当前文库相关的结果。

效果：

- 标签用于横向标注方法、主题、任务、状态或用户自定义分类。

验收标准：

- 标签变更后，列表、详情、筛选和批量操作中的标签状态保持一致。

### 7.5 论文详情与笔记

需求：

- 详情面板应展示论文标题、作者、文件路径或来源、阅读入口、加星入口。
- 如果论文有增强元数据，应展示中文标题、中文摘要、贡献总结、中文 Abstract、英文 Abstract 和生成标签。
- 用户可在详情中调整论文所属文件夹。
- 用户可在详情中调整论文标签。
- 用户可为论文创建、编辑和删除笔记。
- 笔记应支持标题和正文。

效果：

- 用户在不进入阅读器的情况下，也能完成论文归档、元数据检查和阅读记忆记录。

验收标准：

- 笔记保存后再次选中论文仍可看到。
- 未选中论文时，详情区应显示空状态而不是错误状态。

### 7.6 PDF 阅读器

需求：

- 用户可阅读 PDF。
- 用户可翻上一页、下一页。
- 用户可缩小、放大、适配宽度。
- 工具栏应显示当前页码和总页数。
- 用户可在同一阅读会话中切换论文标签。
- 用户可向当前会话添加文库中的其他论文。
- 用户可从当前会话移除论文。
- 用户可选中 PDF 文本，并把选区作为下一轮对话来源。
- 用户点击回答引用时，PDF 应跳转到对应页面和原文区域。
- 从引用跳转后，用户应能返回跳转前位置。
- 用户可打开 PDF 分屏，用于预览内部链接或并排查看同一论文不同位置。
- 分屏应能关闭，并在切换论文时回到合理状态。

效果：

- PDF 阅读器应让用户保持阅读上下文，不因对话、引用跳转或内部链接而迷失位置。

验收标准：

- 选区、引用跳转和返回来源都能围绕真实 PDF 位置工作。
- 多论文会话中切换论文不会丢失会话消息。
- 没有选中论文时应显示清晰空状态。

### 7.7 阅读会话与聊天

需求：

- 每篇或每组论文都可拥有一个或多个会话。
- 用户可新建会话。
- 用户可重命名会话。
- 用户可从会话下拉菜单切换当前会话。
- 会话应显示消息历史。
- 用户消息应与 Agent 消息有清晰视觉区分。
- 用户可输入自由问题并发送。
- 用户可在 Agent 运行中停止当前运行。
- 输入框高度可由用户调整，并在后续使用中保持。
- 当前 PDF 选区应以“回复原文”的形式显示在输入区上方，用户可移除该来源。
- 用户可使用快捷 Prompt 发起常见问题。
- 用户可选择聊天 Agent runtime。
- 用户可为聊天选择模型覆盖项和推理强度。
- 用户可刷新 Agent 状态和模型列表。
- Agent 运行时应展示进行中状态、工具调用、思考/回答/用量等可见进度。
- Agent 回答应支持 Markdown、公式、列表、代码块和引用。
- Agent 回答中的引用应可点击并跳回 PDF。
- 如果 Agent 运行失败，用户应能 Retry 或 New Session。

效果：

- 对话区不是普通聊天框，而是带来源、运行可见性和论文定位能力的研究协作面板。

验收标准：

- 发送消息后，消息历史更新，运行状态可见。
- 停止运行后，用户可继续输入或重新发送。
- 引用点击后，阅读器跳到对应原文。
- 失败消息提供明确下一步，不让用户卡在不可恢复状态。

### 7.8 Agent 终端

需求：

- 会话面板应提供 Terminal 标签。
- 用户可选择支持终端的 Agent runtime。
- 用户可启动和停止终端。
- 用户可调整终端列数和行数。
- 用户可向终端发送输入。
- 终端应显示当前 runtime、日志路径或可检查位置。

效果：

- 高级用户可在当前论文会话工作区中直接操作 Agent 终端，并保持与阅读上下文关联。

验收标准：

- 终端启动、输入、停止和尺寸调整都有明确状态反馈。
- 不支持终端的 runtime 不应让用户误以为可用。

### 7.9 会话笔记

需求：

- 会话面板应提供 Notes 标签。
- 用户可针对当前论文查看笔记列表。
- 用户可新建、编辑、删除笔记。
- 用户可在笔记列表中选择笔记查看或编辑。
- 未选中论文时，应提示用户先选择论文。

效果：

- 用户可在阅读和对话旁边记录自己的研究判断，避免笔记从论文上下文中脱离。

验收标准：

- 笔记创建、编辑和删除后，当前论文笔记列表同步更新。

### 7.10 生成图像展示

需求：

- 当 Agent 消息包含本地生成图片时，聊天区应展示图片缩略图。
- 用户可点击缩略图进入应用内预览。
- 预览应支持放大、缩小、平移或可读查看。
- 用户可关闭预览并回到原聊天和阅读上下文。
- 生成图像应作为会话产物被保留并可再次查看。

效果：

- 图像生成结果成为研究会话的一部分，而不是散落在外部文件夹中的不可见产物。

验收标准：

- 图片缩略图可见、可点击、可关闭预览。
- 预览不会破坏当前阅读器和会话状态。

### 7.11 arXiv Discover

需求：

- 用户可选择日期范围。
- 用户可使用常用快捷日期范围。
- 用户可输入关键词、方法、作者或 arXiv ID。
- 用户可选择主要 arXiv 类别。
- 用户可选择本地相似度来源。
- 用户可开始搜索，也可停止搜索。
- 结果应以响应式卡片网格展示。
- 卡片应展示论文标题、作者、类别、日期、摘要或增强摘要、标签、资源链接、相似度、保存状态和打开动作。
- 用户可按结果类别筛选。
- 用户可按处理状态筛选：全部、已处理、未处理、失败。
- 用户可按文库状态筛选：全部、新论文、已在文库。
- 用户可筛选有代码或项目链接的论文。
- 用户可按相似度等级筛选：高、中、低、无相似度。
- 用户可按生成标签筛选。
- 当前筛选条件应以可移除 chip 展示。
- 用户可对当前可见结果批量处理。
- 处理动作至少包括：翻译、总结、下载 PDF 并生成缩略图。
- 处理时用户可选择模型和推理强度。
- 处理、搜索和缓存过程应显示进度。
- 用户可停止搜索、处理或 PDF 缓存。
- 用户可预览论文图像或 PDF 缩略图。
- 用户可从卡片保存论文到文库，并选择已有文件夹或创建新文件夹。
- 用户可直接打开 arXiv 论文进入阅读器。
- 页面应尽量恢复用户上次浏览位置。

效果：

- Discover 应帮助用户快速扫新论文、判断价值、发现相关论文并进入阅读。

验收标准：

- 搜索结果、筛选数量、处理状态和保存状态保持一致。
- 保存到文库后，卡片显示已在文库，文库中可找到该论文。
- 批量处理失败的结果可通过状态筛选被找到。

### 7.12 arXiv 搜索

需求：

- 用户可输入 arXiv 查询语法。
- 用户可选择排序维度：相关性、提交日期、更新日期。
- 用户可切换升序/降序。
- 用户可填写起始年份和结束年份。
- 用户可设置必须包含的 arXiv 类别；也可选择不限制类别。
- 用户可按搜索结果类别筛选。
- 用户可按文库状态筛选。
- 用户可对搜索结果执行保存、打开、预览和处理动作。

效果：

- 搜索页面向目标明确的检索，Discover 面向近期浏览和发现，两者共享结果后续动作。

验收标准：

- 查询、年份、类别和排序共同影响搜索结果。
- 用户清除筛选 chip 后，结果区域立即反映新的条件。

### 7.13 论文卡片与资源链接

需求：

- 论文卡片应展示 arXiv ID，并允许用户识别来源。
- 卡片应显示是否已保存到文库。
- 卡片应提供 Add 和 Open 动作。
- 如果论文有关联项目、代码、Hugging Face 或其他资源链接，应在卡片中显示可点击入口。
- 卡片应展示相似度分数或无相似度状态。
- 卡片应在下载或打开时显示忙碌状态和进度。

效果：

- 用户不需要打开详情页，也能完成“判断、跳转资源、保存、阅读”的主要动作。

验收标准：

- 外部资源链接点击后打开对应资源。
- 已保存论文不应继续表现为普通新论文。

### 7.14 最近会话

需求：

- Recent Conversations 应展示最近研究会话列表。
- 每个会话应显示标题、相对更新时间、单论文/多论文状态和简要论文信息。
- 用户可选中会话查看详情。
- 详情应展示会话包含的论文。
- 用户可从列表或详情打开会话。

效果：

- 用户可以把 Episteme 当成持续研究空间，历史上下文可恢复。

验收标准：

- 打开历史会话后，Reader、论文集合、消息和会话选择状态一致。

### 7.15 全局语言

需求：

- 用户可在设置中选择应用语言模式。
- 语言模式应影响应用界面、Discover 展示语言和默认 Agent 提示语气。

效果：

- 用户可在中文阅读工作流和英文论文原文之间保持顺畅切换。

验收标准：

- 切换语言后，主要界面文本和可生成内容的默认语言偏好同步更新。

### 7.16 聊天外观

需求：

- 用户可选择聊天字体。
- 用户可调整消息字体大小。
- 用户可调整输入框字体大小。
- 设置页应提供外观预览。
- 用户可恢复默认外观。

效果：

- 长时间阅读和对话时，用户可按屏幕、视力和个人习惯调整可读性。

验收标准：

- 保存外观设置后，Reader 聊天区立即使用新设置。
- 默认按钮能恢复出厂样式。

### 7.17 arXiv Feed 偏好

需求：

- 用户可配置默认 arXiv 类别列表。
- 用户可保存类别设置。
- 用户可刷新 arXiv 日期和 feed。
- 设置页应显示当前缓存日期或无缓存状态。

效果：

- 用户可把 Discover 调整为自己关注的研究领域。

验收标准：

- 保存类别后，Discover 后续使用新的类别偏好。

### 7.18 本地排序与相似度

需求：

- 用户可配置用于本地排序的标签白名单。
- 用户可配置标签黑名单。
- 用户可选择用于相似度来源的文库文件夹。
- 设置页应展示已选择的相似度来源数量。

效果：

- Discover 结果能优先贴近用户已有研究收藏和关注主题。

验收标准：

- 修改排序/相似度偏好后，Discover 的相似度筛选和排序依据可体现新偏好。

### 7.19 Codex / Agent 增强

需求：

- 用户可设置打开 arXiv 论文时自动增强。
- 用户可设置保存到文库时自动增强。
- 用户可选择 Discover 处理使用的模型。
- 用户可输入自定义模型覆盖。
- 用户可选择推理强度。
- 用户可设置并发处理数量。
- 用户可刷新可用模型。
- 用户可编辑阅读对话的系统 Prompt。
- 用户可恢复默认系统 Prompt。

效果：

- 用户能控制增强质量、速度、成本和提示风格。

验收标准：

- 保存设置后，后续处理和聊天使用新的用户偏好。
- 空系统 Prompt 不应被保存为有效 Prompt。

### 7.20 Agent runtime 管理

需求：

- 设置页应展示可用 Agent runtime。
- 用户可分别选择聊天 runtime 和探索增强 runtime。
- 用户可启用或禁用每个 runtime。
- 用户可查看每个 runtime 的可用性、版本、路径、认证状态或阻塞原因。
- 用户可为 runtime 设置模型覆盖。
- 对需要 provider 的 runtime，用户可设置 provider 覆盖。
- 用户可选择 runtime 的 MCP 模式。
- 用户可刷新 runtime 检查状态。

效果：

- 用户可以把 Episteme 作为统一阅读前端，选择不同 Agent 来完成聊天和增强任务。

验收标准：

- 被禁用或不可用的 runtime 不应被误用。
- runtime 状态变化后，聊天区和设置页状态一致。

### 7.21 Embedding provider

需求：

- 用户可启用或禁用 embedding 相似度。
- 用户可设置 base URL。
- 用户可设置 API key。
- 用户可设置模型名。
- 用户可测试 provider 是否可用。
- 测试结果应以状态文本展示。

效果：

- 用户可以接入 OpenAI-compatible embedding 服务，让本地文库与 arXiv 结果形成相似度关联。

验收标准：

- 测试成功或失败都应给出明确状态。
- 留空 API key 时，不应意外清除已保存密钥，除非用户明确要求。

### 7.22 快捷 Prompt

需求：

- 用户可查看当前快捷 Prompt 列表。
- 用户可新增快捷 Prompt。
- 用户可编辑快捷 Prompt 标题和内容。
- 用户可删除快捷 Prompt。
- 用户可调整快捷 Prompt 顺序。
- 阅读器聊天输入区可直接使用快捷 Prompt 发起提问。

效果：

- 用户可以把常用研究问题固化为个人阅读模板。

验收标准：

- 新增或编辑后的 Prompt 在聊天区 Quick Prompt 菜单可用。
- 空标题或空内容不应被保存。

### 7.23 保存组织规则

需求：

- 用户可选择从 arXiv 保存论文时的默认文件夹组织规则。
- 可选规则应包括：按主类别、按第一个标签、按 feed 日期、平铺文库。
- 设置页应展示文库根位置，并支持在系统文件管理器中定位。

效果：

- 用户可以控制批量保存论文时的自动归档方式，减少手动整理成本。

验收标准：

- 保存规则变化后，新保存论文按当前规则归档。

### 7.24 缓存管理

需求：

- 设置页应展示缓存根位置。
- 设置页应展示 arXiv 缓存、缩略图等空间占用摘要。
- 用户可刷新缓存大小。
- 用户可清理 arXiv 缓存。
- 清理前必须确认，并说明会删除 feed JSON、临时 PDF 和未保存的已打开 arXiv 论文。

效果：

- 用户能理解并管理本地缓存，不把缓存误认为正式文库。

验收标准：

- 清理缓存不会删除已正式保存到文库的论文。
- 清理后缓存大小摘要可刷新。

### 7.25 Episteme MCP / 外部 Agent 能力

需求：

- 用户可在设置中启用或禁用应用内 Agent 使用 Episteme MCP。
- 用户可查看 MCP 当前状态。
- 用户可安装或更新本地 Codex 插件。
- 外部 Agent 应能读取当前应用上下文、论文列表、论文元数据、全文、页面、段落、锚点、笔记、文件夹、标签、最近会话、会话消息和会话工作区信息。
- 外部 Agent 应能通过受控工具执行论文导入、搜索、元数据更新、加星、删除、去重、文件夹归属、标签、笔记、锚点、引用解析、打开论文、跳页、跳锚点、监听文件夹和会话查询等动作。
- 破坏性或批量动作应要求明确确认。
- 外部 Agent 应能读取会话工作区清单、运行时信息和 Prompt 契约，以便在本地会话中产出可引用、可检查的回答。

效果：

- Episteme 不只是一个封闭应用，也能成为 Codex 或其他 Agent 的论文上下文来源和受控操作界面。

验收标准：

- 安装/更新后，外部 Agent 可以发现 Episteme 插件和资源。
- 外部 Agent 修改应用状态后，应用侧状态可被重新读取并验证。
- 外部 Agent 对论文事实的回答应能基于全文、段落、锚点或会话工作区，而不是凭空回答。

## 8. 用户可见状态与反馈

应用应覆盖以下状态：

- 空文库：提示用户导入 PDF、添加 arXiv 论文或配置监听文件夹。
- 空搜索结果：明确当前筛选/查询没有结果。
- 加载中：Discover、搜索、处理、缓存、打开 arXiv 论文、Agent 运行应有进度或忙碌状态。
- 可取消任务：长耗时搜索、处理、缓存和 Agent 运行应允许停止。
- 处理失败：失败结果应可筛选，失败消息应给出重试或新会话选择。
- 未选择论文：Reader、笔记、详情等区域应显示可理解空状态。
- 破坏性操作确认：删除论文、删除文件夹、删除标签、移除监听文件夹、清理缓存等动作应要求确认。
- 保存状态：arXiv 卡片应区分新论文和已在文库论文。
- 运行时状态：Agent runtime 应展示 ready、warning、blocked、checking 等用户可理解状态。

## 9. 非功能性用户体验要求

- 本地优先：用户的论文、笔记、会话和生成物应尽量保留在本机可控空间。
- 上下文连续：从文库到阅读器、从引用到原文、从历史会话到当前阅读，都应尽量保留上下文。
- 可检查：Agent 运行、引用、生成物和工作区应能被用户检查。
- 可恢复：会话、阅读位置、Discover 浏览位置和常用设置应在关闭或切换后尽量恢复。
- 可读性：长文本、Markdown、公式、代码块、引用和图片预览应适合长时间阅读。
- 明确边界：正式文库、临时 arXiv 缓存、生成物、会话工作区和外部 Agent 操作应在用户效果上可区分。

## 10. 重构验收清单

重构版本至少应通过以下端到端验收：

1. 导入 PDF，创建文件夹和标签，给论文加星，搜索并筛选后打开阅读。
2. 在 PDF 中选中文本，发送问题，看到 Agent 运行状态、回答和可点击引用，并能跳回原文。
3. 在同一会话中加入第二篇论文，进行多论文提问，并切换论文标签。
4. 在文库中多选论文，分别完成批量阅读、批量聊天、复制到文件夹、添加标签和删除确认。
5. 在 Discover 中按日期和类别搜索，处理当前结果，筛选已处理/失败/新论文，保存一篇到文库。
6. 在搜索页用 arXiv 查询、年份、类别和排序找到论文，并执行保存和打开。
7. 在最近会话页打开历史会话，确认论文集合、消息、笔记和阅读上下文恢复。
8. 调整聊天字体、快捷 Prompt、Agent runtime、模型/推理强度、embedding provider、保存组织规则和缓存设置，确认后续工作流使用新偏好。
9. 生成或展示一张本地图片，确认聊天缩略图和应用内可缩放预览可用。
10. 安装或更新 Episteme MCP 插件，外部 Agent 可读取当前上下文，并能通过受控动作更新笔记或跳转阅读器。

## 11. 后续产品问题

以下问题不阻塞重构，但需要在新框架版本定稿前确认：

- 是否继续保持“本地优先单机工作台”为主定位，还是加入账号、云同步或团队协作。
- 外部 Agent 能力是否要从 Codex 插件扩展为多 Agent 通用协议。
- Discover 的增强结果是否需要更强的人工编辑和质量标注。
- 论文笔记是否需要独立知识库视图，而不仅是论文详情和会话侧栏。
- 多论文会话是否需要显式任务类型，例如综述、对比、审稿、复现计划。
